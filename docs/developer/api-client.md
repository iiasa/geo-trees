# API Client

## 1. Overview

The TypeScript API client lives in `frontend/src/infrastructure/api/` and is **auto-generated** from the backend's OpenAPI specification. It is produced by [hey-api](https://heyapi.dev/) (`@hey-api/openapi-ts`) and regenerated by running `pnpm generate-api` from the `frontend/` directory.

**Never edit files in `src/infrastructure/api/` manually.** Every file in that directory carries a `.gen.ts` suffix (or is inside a generated subdirectory) and will be overwritten the next time `pnpm generate-api` runs.

---

## 2. What Gets Generated

Running `pnpm generate-api` populates `src/infrastructure/api/` with the following:

| File / path | Purpose |
|---|---|
| `types.gen.ts` | TypeScript request and response types inferred from the OpenAPI spec |
| `sdk.gen.ts` | Typed fetch functions — one per API endpoint |
| `zod.gen.ts` | Zod schemas for runtime validation, with `z.infer<>` types |
| `@tanstack/react-query.gen.ts` | `*Options` query-options functions, `*QueryKey` key functions, and `*Mutation` mutation-options functions |
| `client.gen.ts` / `client/` | The underlying hey-api fetch client and configuration types |
| `core/` | Internal helpers (auth, body serialization, path/query serialization, SSE) |
| `index.ts` | Re-exports for convenient top-level imports |

---

## 3. How to Use — Queries

Import the generated `*Options` function and pass it directly to TanStack Query's `useQuery`. The returned `data` is fully typed.

Real example from `frontend/src/features/users/components/users-list.tsx`:

```typescript
import { getApiIdentityUsersOptions } from "@/infrastructure/api/@tanstack/react-query.gen";
import { useQuery } from "@tanstack/react-query";

function UsersList() {
  const { data, isLoading, isError, error } = useQuery(
    getApiIdentityUsersOptions({
      query: {
        MaxResultCount: 10,
        SkipCount: 0,
      },
    })
  );

  const users = data?.items ?? [];
  const totalCount = data?.totalCount ?? 0;
  // ...
}
```

The generated `*Options` functions also expose a companion `*QueryKey` function that returns the stable query key, useful when you need to invalidate or refetch a specific query:

```typescript
import {
  getApiIdentityUsersOptions,
  getApiIdentityUsersQueryKey,
} from "@/infrastructure/api/@tanstack/react-query.gen";

queryClient.invalidateQueries({
  queryKey: getApiIdentityUsersQueryKey(queryOptions),
});
```

## 4. How to Use — Mutations

Import the generated `*Mutation` function and spread it into `useMutation`. Combine with `useQueryClient` to invalidate stale queries on success.

Real example based on `frontend/src/features/users/components/users-list.tsx`:

```typescript
import {
  postApiIdentityUsersMutation,
  getApiIdentityUsersQueryKey,
} from "@/infrastructure/api/@tanstack/react-query.gen";
import { useMutation, useQueryClient } from "@tanstack/react-query";

function CreateUserForm() {
  const queryClient = useQueryClient();

  const createUserMutation = useMutation({
    ...postApiIdentityUsersMutation(),
    onSuccess: () => {
      queryClient.invalidateQueries({
        queryKey: getApiIdentityUsersQueryKey(queryOptions),
      });
    },
  });

  const handleSubmit = () => {
    createUserMutation.mutate({
      body: {
        userName: "alice",
        email: "alice@example.com",
        password: "...",
        isActive: true,
        lockoutEnabled: false,
      },
    });
  };

  return <button onClick={handleSubmit}>Create User</button>;
}
```

For mutations that target a specific resource, pass `path` parameters alongside `body`:

```typescript
deleteUserMutation.mutateAsync({
  path: { id: userId },
});
```

## 5. Regenerating the Client

Regenerate whenever the backend API changes — new endpoints, renamed fields, changed request or response shapes.

```bash
# The backend must be running first, then from the frontend directory:
pnpm generate-api
```

This command:
1. Fetches the OpenAPI spec from `VITE_OPENAPI_SPEC_URL` (default: `https://localhost:44324/swagger/v1/swagger.json`)
2. Cleans `src/infrastructure/api/`
3. Regenerates all types, SDK functions, Zod schemas, and TanStack Query helpers

After regeneration, TypeScript will immediately surface any breaking changes at compile time — run `pnpm typecheck` to check.

## 6. Client Configuration

The generated fetch client is configured at runtime in `frontend/src/hey-api.ts`:

```typescript
// frontend/src/hey-api.ts
import { API_CONSTANTS } from "@/shared/constants";
import type { Config } from "@/infrastructure/api/client";

export const createClientConfig = (config?: Config) => ({
  ...config,
  baseUrl: API_CONSTANTS.PROXY_PATH,
});
```

`API_CONSTANTS.PROXY_PATH` resolves to `/api/proxy` (configurable via `VITE_API_PROXY_PATH`).

**Why `/api/proxy` and not the backend directly?**

All API calls from the hey-api client are sent to the frontend's own `/api/proxy` server route. That route forwards each request to the real backend (`VITE_API_BASE_URL`) and automatically attaches the user's session credentials as auth headers. This means:

- No CORS issues — browser-to-frontend and frontend-to-backend are same-origin or server-side respectively
- Auth is applied transparently — components never handle tokens directly
- The backend URL never leaks to the browser

## 7. hey-api Configuration

The code-generation behaviour is controlled by `frontend/hey-api.config.ts`:

```typescript
// frontend/hey-api.config.ts
import { defineConfig } from '@hey-api/openapi-ts';

export default defineConfig({
  input: process.env.VITE_OPENAPI_SPEC_URL || 'https://localhost:44324/swagger/v1/swagger.json',
  output: {
    path: 'src/infrastructure/api',
    clean: true,
  },
  client: 'fetch',
  plugins: [
    {
      name: '@hey-api/client-fetch',
      runtimeConfigPath: '../../hey-api', // Points to src/hey-api.ts
    },
    {
      name: 'zod',
      types: { infer: true }, // Generate z.infer<> types
    },
    {
      name: '@tanstack/react-query',
      queryOptions: true,   // Generate *Options functions
      queryKeys: true,      // Generate *QueryKey functions
      mutationOptions: true, // Generate *Mutation functions
    },
  ],
});
```

Key points:

- **Input**: the backend's Swagger JSON endpoint. Override with the `VITE_OPENAPI_SPEC_URL` environment variable if your backend runs on a different port.
- **Output**: `src/infrastructure/api/`, cleaned on every run.
- **`@hey-api/client-fetch`**: generates the fetch client; `runtimeConfigPath` points to `src/hey-api.ts` so the `baseUrl` override is baked in at generation time.
- **`zod`**: generates Zod schemas alongside TypeScript types, enabling runtime validation of API responses.
- **`@tanstack/react-query`**: generates the `*Options`, `*QueryKey`, and `*Mutation` helpers that components import directly.
