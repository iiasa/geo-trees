# Database Migrations

GeoTrees uses Entity Framework Core for database migrations.

## Running Migrations

```bash
# Via Makefile (recommended)
make be-migrate

# Equivalent command
cd backend/IIASA.GeoTrees && dotnet run -- --migrate-database
```

## When to Run Migrations

- **First-time setup**: Before running the backend for the first time
- **After pulling updates**: If the update includes new migration files in `backend/IIASA.GeoTrees/Migrations/`
- **Before deploying a new version**: Always run migrations before deploying

## Creating New Migrations (for developers)

After adding or modifying an entity:

```bash
cd backend/IIASA.GeoTrees
dotnet ef migrations add DescriptiveMigrationName
```

Migration files are created in `backend/IIASA.GeoTrees/Migrations/`. Commit these files — they are part of the codebase.

## Reverting Migrations

```bash
cd backend/IIASA.GeoTrees
# Revert to a specific migration
dotnet ef database update PreviousMigrationName

# Remove the latest migration (only if not applied to production)
dotnet ef migrations remove
```

## Migration Files
Migration files live in `backend/IIASA.GeoTrees/Migrations/`. These are auto-generated by EF Core — do not edit them manually unless you know what you're doing.
