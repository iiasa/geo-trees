# Docker

GeoTrees ships with Dockerfiles and Docker Compose configurations for both the frontend and backend.

## Full Stack with Docker Compose

The Makefile `docker-up` target starts both the frontend and backend (including Postgres) together:

```bash
make docker-up    # Start full stack (frontend + backend + Postgres)
make docker-down  # Stop everything
```

`docker-up` is a shortcut that runs `docker-be-up` followed by `docker-fe-up`. Tear down runs in reverse order.

## Frontend Only

```bash
make docker-fe-up    # cd frontend && docker-compose up --build -d
make docker-fe-down  # cd frontend && docker-compose down
```

The frontend Docker Compose (`frontend/docker-compose.yml`) starts a single `app` service on port **3000** with `NODE_ENV=production`. A health check polls `GET /api/health` every 30 seconds.

## Backend Only (includes Postgres)

```bash
make docker-be-up    # cd backend/etc/docker && docker-compose up --build -d
make docker-be-down  # cd backend/etc/docker && docker-compose down
```

The backend Docker Compose (`backend/etc/docker/docker-compose.yml`) starts three services:

| Service | Image | Ports | Notes |
|---------|-------|-------|-------|
| `geotrees-web` | `iiasa/geotrees:latest` | `44324:8081` (HTTPS), `8080` (HTTP) | Main backend API |
| `db-migrator` | `iiasa/geotrees:latest` | — | Runs `--migrate-database` then exits |
| `postgres-db` | `postgres:14.1` | `5432:5432` | Waits for health check before app starts |

All services share a custom bridge network (`geotrees-network`). The app and migrator both wait for Postgres to be healthy before starting.

## Building Backend Images Manually

1. Navigate to `backend/etc/build/`
2. Run `build-images-locally.ps1` (PowerShell)
3. The default image tag is `latest` — edit the script to change it

## Running Backend Docker Compose via Scripts

Scripts in `backend/etc/docker/` provide a convenience wrapper around Docker Compose:

- **`run-docker.ps1`** — generates dev certificates (if they do not exist yet) with `dotnet dev-certs`, then starts the containers in detached mode
- **`stop-docker.ps1`** — stops and removes the running containers

> The dev certificate generated by `run-docker.ps1` is only valid for `localhost`. For a real domain, use Let's Encrypt or a CA-issued certificate.

## Dockerfiles

### `frontend/Dockerfile`

Multi-stage build using `node:24-alpine`:

- **Builder stage**: installs pnpm, runs `pnpm install --frozen-lockfile`, then `pnpm run build`
- **Production stage**: copies only `.output/`, `node_modules/`, and `package.json` from the builder; runs as a non-root user (`tanstack`, uid 1001); uses `dumb-init` for proper signal handling
- Exposes port **3000**
- Sets `NODE_ENV=production` and `PORT=3000`
- Health check: `GET http://localhost:3000/api/health`
- Entrypoint: `node .output/server/index.mjs`

### `backend/IIASA.GeoTrees/Dockerfile`

Multi-stage build using `mcr.microsoft.com/dotnet/aspnet:10.0` (runtime) and `mcr.microsoft.com/dotnet/sdk:10.0` (build):

- **Build stage**: restores NuGet packages, copies source, builds in Release configuration
- **Publish stage**: installs Node.js and the ABP CLI, runs `abp install-libs`, then publishes the app
- **Final stage**: copies the published output, runs as the `app` user, exposes port **8080** (`http://+:8080`)
- **Entrypoint script** (`entrypoint.sh`): if `openiddict.pfx` does not exist, auto-generates a self-signed RSA-2048 certificate using `openssl` and the `AuthServer__CertificatePassPhrase` environment variable; then runs `--migrate-database` and starts the app

### `backend/IIASA.GeoTrees/Dockerfile.local`

Lightweight image for local development builds using `mcr.microsoft.com/dotnet/sdk:10.0`:

- Copies a pre-built publish output from `bin/Release/net10.0/publish/`
- Exposes ports **8080** and **8081**
- No entrypoint script — starts directly with `dotnet IIASA.GeoTrees.dll`
- Used by `backend/etc/docker/docker-compose.yml` (the `make docker-be-up` target)

## Backend Docker Compose — Supporting Services

The backend development Docker Compose (`backend/docker-compose.yml`, used by `make docker-be-up`) also includes:

| Service | Image | Ports | Purpose |
|---------|-------|-------|---------|
| `postgres` | `postgis/postgis:16-3.5` | `5432:5432` | PostgreSQL with PostGIS extension |
| `adminer` | `adminer` | `8080:8080` | Web-based database admin UI |
| `mailpit` | `axllent/mailpit` | `8025` (UI), `1025` (SMTP) | Local email capture for testing |

> Note: `postgis/postgis:16-3.5` is used here (with PostGIS), while the backend's `etc/docker/docker-compose.yml` uses plain `postgres:14.1`. Ensure your production database has the PostGIS extension installed if geospatial features are required.
